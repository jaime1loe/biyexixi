# 最近活动加载失败（500错误）修复说明

## 问题描述

前端Dashboard页面加载最近活动时出现500错误：
```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
加载最近活动失败: AxiosError: Request failed with status code 500
```

## 问题根源

**Schema定义与数据库模型不匹配**

在 `backend/app/schemas.py` 中，`ActivityLogResponse` 的字段定义与 `ActivityLog` 模型的字段不匹配：

### 错误的Schema定义（修复前）
```python
class ActivityLogResponse(BaseModel):
    """活动日志响应模型"""
    id: int
    user_id: int
    activity_type: str              # ❌ 错误：应该是 action_type
    description: str                 # ❌ 错误：模型中没有此字段
    ip_address: Optional[str] = None
    user_agent: Optional[str] = None # ❌ 错误：模型中没有此字段
    created_at: datetime
```

### 正确的数据库模型字段
```python
class ActivityLog(Base):
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, nullable=False)
    action_type = Column(String(50), nullable=False)   # ✓ action_type
    target_type = Column(String(50))                    # ✓ target_type
    target_id = Column(Integer)                         # ✓ target_id
    target_name = Column(String(200))                  # ✓ target_name
    details = Column(Text)                              # ✓ details
    ip_address = Column(String(50))                     # ✓ ip_address
    created_at = Column(DateTime)
```

## 修复内容

已将 `ActivityLogResponse` schema 修改为：

```python
class ActivityLogResponse(BaseModel):
    """活动日志响应模型"""
    id: int
    user_id: int
    username: Optional[str] = None       # 新增：用户名
    user_role: Optional[str] = None      # 新增：用户角色
    action_type: str                      # ✓ 修正：activity_type -> action_type
    target_type: Optional[str] = None    # 新增：目标类型
    target_id: Optional[int] = None      # 新增：目标ID
    target_name: Optional[str] = None    # 新增：目标名称
    details: Optional[str] = None        # 新增：详细信息
    ip_address: Optional[str] = None
    created_at: datetime
```
