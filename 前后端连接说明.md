# 前后端连接配置说明

## 系统架构

```
前端 (Vue 3 + Vite)
    ↓ HTTP请求 (通过代理)
后端 (FastAPI + Python)
    ↓ SQL查询
数据库 (MySQL)
```

## 配置详情

### 1. 后端配置

#### 数据库配置 (`backend/.env`)
```env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=qa_system
DB_USER=root
DB_PASSWORD=12345678
```

#### 服务器配置 (`backend/.env`)
```env
HOST=0.0.0.0

```

### 2. 前端配置

#### API代理配置 (`frontend/vite.config.ts`)
```typescript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 代理到后端
      changeOrigin: true
    }
  }
}
```

#### API基础URL (`frontend/src/api/index.ts`)
```typescript
const request = axios.create({
  baseURL: '/api',  // 使用代理路径
  timeout: 30000
})
```

## 工作原理

### 请求流程

1. **前端发送请求**
   - 前端调用 API：`authApi.login({ username, password })`
   - 实际请求：`POST /api/auth/login`

2. **Vite开发服务器处理**
   - 拦截 `/api/*` 路径的请求
   - 转发到 `http://localhost:8000/api/auth/login`

3. **后端FastAPI处理**
   - 接收请求：`POST /api/auth/login`
   - 处理业务逻辑
   - 查询MySQL数据库
   - 返回响应

4. **前端接收响应**
   - Vite代理接收后端响应
   - 转发给前端
   - 更新UI显示

### 认证流程

1. **登录**
   ```
   前端 → POST /api/auth/login
   后端 → 验证用户密码（从数据库）
   后端 → 返回JWT token
   前端 → 存储到 localStorage
   ```

2. **访问受保护资源**
   ```
   前端 → 请求时携带 Authorization: Bearer {token}
   后端 → 验证token有效性
   后端 → 返回受保护数据
   ```

## 启动步骤

### 方法1：使用一键启动脚本（推荐）

双击运行 `启动系统.bat`，脚本会自动：
1. 启动后端服务（端口8000）
2. 启动前端服务（端口5173）
3. 打开浏览器访问前端

### 方法2：手动启动

#### 启动后端

```bash
# 进入后端目录
cd d:/毕业设计/backend

# 启动后端服务
python main.py

# 或者使用uvicorn
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

看到以下输出表示启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

打开新的命令窗口：

```bash

# 安装依赖（首次运行）
npm install --registry=https://registry.npmmirror.com

npm run dev
```

看到以下输出表示启动成功：

## 常见问题

### 问题1：前端无法连接后端

**症状**：前端显示"请求失败"或控制台报错

**排查步骤**：

1. 检查后端是否启动
   ```bash
   # 访问 http://localhost:8000/docs
   # 或 curl http://localhost:8000/health
   ```

2. 检查端口是否正确
   - 后端：8000
   - 前端：5173

3. 检查代理配置
   - `frontend/vite.config.ts` 中 `target` 应为 `http://localhost:8000`

4. 查看浏览器控制台错误
   - 按 F12 → Console 标签
   - 查看红色错误信息

### 问题2：数据库连接失败

**症状**：后端启动时报错"Access denied"或"Can't connect"

**解决方法**：

1. 确认MySQL密码为 `12345678`
   - 编辑 `backend/.env`
   - 设置 `DB_PASSWORD=12345678`

2. 确认MySQL服务已启动
   ```bash
   net start mysql
   ```

3. 确认数据库已创建
   ```sql
   CREATE DATABASE IF NOT EXISTS qa_system;
   ```

4. 测试连接
   ```bash
   mysql -u root -p12345678 -e "SHOW DATABASES;"
   ```

### 问题3：端口被占用

**症状**：启动时报错"Address already in use"

**解决方法**：

1. 查找占用进程
   ```bash
   # 查看端口8000
   netstat -ano | findstr :8000

   # 查看端口5173
   netstat -ano | findstr :5173
   ```

2. 结束进程
   ```bash
   taskkill /F /PID <进程ID>
   ```

3. 或修改端口
   - 后端：修改 `backend/.env` 中的 `PORT`
   - 前端：修改 `frontend/vite.config.ts` 中的 `port`

### 问题4：跨域问题（CORS）

**症状**：浏览器控制台显示 CORS 错误

**检查**：

后端已配置 CORS（`backend/main.py`）：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

如有问题，重启后端服务。

## API接口文档

### 认证接口

- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息

### 问答接口

- `POST /api/questions/` - 提问
- `GET /api/questions/` - 获取问题列表
- `GET /api/questions/{id}` - 获取问题详情

### 知识库接口

- `POST /api/knowledge/` - 添加知识
- `GET /api/knowledge/` - 获取知识列表
- `DELETE /api/knowledge/{id}` - 删除知识

### 统计接口

- `GET /api/statistics/overview` - 数据概览
- `GET /api/statistics/daily` - 每日统计

完整API文档：http://localhost:8000/docs

## 数据库表结构

```sql
-- 用户表
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    hashed_password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    real_name VARCHAR(50),
    student_id VARCHAR(20),
    role VARCHAR(20) DEFAULT 'student',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 问题表
CREATE TABLE questions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    question TEXT NOT NULL,
    answer TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 知识库表
CREATE TABLE knowledge (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    category VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 反馈表
CREATE TABLE feedbacks (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    question_id INT,
    rating INT,
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```
