# 排课管理空白页面修复说明

## 问题描述
排课管理页面显示空白，无任何内容。

## 问题原因

### 1. 数据访问错误
`frontend/src/api/index.ts`的响应拦截器会自动提取`response.data`：
```typescript
request.interceptors.response.use(
  (response) => {
    return response.data  // 已经提取了
  },
  ...
)
```

但`ScheduleManagement.vue`中仍然尝试访问`response.data`，导致访问到undefined。

### 2. 后端API不完整
`backend/app/routers/schedule.py`只有GET接口，缺少POST、PUT、DELETE接口，导致前端无法创建、更新、删除排课。

### 3. Schema字段不匹配
`backend/app/schemas.py`中的`ScheduleCreate`和`ScheduleUpdate`使用的是`start_time`和`end_time`字段，而数据库模型和前端使用的是`period`（节次）、`week_start`、`week_end`和`class_name`字段。

## 解决方案

### 1. 修复前端数据访问

**文件**: `frontend/src/components/admin/ScheduleManagement.vue`

#### loadSchedules函数
```typescript
// 修改前
const response = await scheduleApi.getSchedules(params)
schedules.value = response.data  // ❌ 错误

// 修改后
const response = await scheduleApi.getSchedules(params)
schedules.value = Array.isArray(response) ? response : []  // ✅ 正确
```

#### loadCourses和loadClassrooms函数
同样修改，确保数据访问正确。

#### checkConflict函数
```typescript
const response = await scheduleApi.getSchedules({...})
const schedules = Array.isArray(response) ? response : []  // 安全处理
```

### 2. 添加后端CRUD接口

**文件**: `backend/app/routers/schedule.py`

#### 新增接口
- `POST /api/schedules` - 创建排课
  - 验证课程和教室存在
  - 检查教室容量
  - 检查时间冲突
  - 记录活动日志

- `PUT /api/schedules/{id}` - 更新排课
  - 验证权限（仅管理员）
  - 验证时间冲突（排除自己）
  - 记录活动日志

- `DELETE /api/schedules/{id}` - 删除排课
  - 验证权限（仅管理员）
  - 记录活动日志

#### 时间冲突检测逻辑
```python
conflict = db.query(Schedule).filter(
    Schedule.classroom_id == schedule_data.classroom_id,
    Schedule.day_of_week == schedule_data.day_of_week,
    Schedule.period == schedule_data.period,
    Schedule.semester == schedule_data.semester,
    Schedule.week_start <= schedule_data.week_end,
    Schedule.week_end >= schedule_data.week_start
).first()
```

周次重叠检测：
- 如果新排课的起始周 <= 已有排课的结束周
- 并且新排课的结束周 >= 已有排课的起始周
- 则认为时间冲突

## 节次定义

系统使用5个节次，每个节次对应固定时间段：

| 节次 | 时间 |
|------|------|
| 第1节 | 08:00-09:40 |
| 第2节 | 10:00-11:40 |
| 第3节 | 14:00-15:40 |
| 第4节 | 16:00-17:40 |
| 第5节 | 19:00-20:40 |

## 数据验证规则

### 创建排课
1. ✅ 必须是管理员
2. ✅ 课程必须存在
3. ✅ 教室必须存在
4. ✅ 教室容量 >= 课程容量
5. ✅ 时间不能冲突

### 更新排课
1. ✅ 必须是管理员
2. ✅ 排课记录必须存在
3. ✅ 如果修改教室，验证新教室存在且容量足够
4. ✅ 时间不能冲突（排除自己）

### 删除排课
1. ✅ 必须是管理员
2. ✅ 排课记录必须存在

## 活动日志记录

所有CRUD操作都会记录到活动日志：
- 创建排课：记录课程名称、教室、时间
- 更新排课：记录课程名称
- 删除排课：记录课程名称、教室、时间
