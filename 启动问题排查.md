# 启动问题排查指南

## 问题1：点击"启动系统.bat"后无法进入登录页面

### 可能原因：

1. **MySQL数据库未启动**
   - 检查MySQL服务是否正在运行
   - 数据库密码是否设置为 `12345678`
   - 数据库 `qa_system` 是否已创建

2. **后端服务启动失败**
   - 查看后端命令窗口的错误信息
   - 检查Python依赖是否完整
   - 确认端口8000未被占用

3. **前端服务启动失败**
   - 查看前端命令窗口的错误信息
   - 检查node_modules是否存在
   - 确认端口5173未被占用

4. **浏览器未正确打开**
   - 手动访问 http://localhost:5173
   - 检查浏览器控制台是否有错误

### 解决步骤：

#### 步骤1：检查MySQL数据库

```bash
# 在命令行执行
mysql -u root -p12345678

# 然后执行
CREATE DATABASE IF NOT EXISTS qa_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 步骤2：检查Python依赖

```bash
cd backend
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 步骤3：检查前端依赖

```bash
cd frontend
npm install --registry=https://registry.npmmirror.com
```

#### 步骤4：手动启动（调试）

**启动后端：**
```bash
cd backend
python main.py
```

查看是否有错误信息，正常应该显示：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

**启动前端：**
```bash
cd frontend
npm run dev
```

查看是否有错误信息，正常应该显示：
```
➜  Local:   http://localhost:5173/
```

#### 步骤5：测试API连接

打开浏览器访问：http://localhost:8000/docs

如果能访问到Swagger文档，说明后端正常。

### 问题2：看到空白页面

**原因：**前端依赖未安装完整

**解决：**
```bash
cd frontend
rmdir /s /q node_modules
npm install --registry=https://registry.npmmirror.com
npm run dev
```

### 问题3：端口被占用

**查看端口占用：**
```bash
# Windows
netstat -ano | findstr :8000
netstat -ano | findstr :5173
```

**结束进程：**
```bash
taskkill /F /PID <进程ID>
```

### 问题4：登录后跳转失败

检查浏览器控制台（F12）：
1. Network标签 - 查看API请求是否成功
2. Console标签 - 查看是否有JavaScript错误

**常见错误：**
- `CORS policy` - 检查后端CORS配置
- `401 Unauthorized` - 检查token是否正确存储
- `404 Not Found` - 检查API路径是否正确

### 问题5：数据库连接失败

检查 `backend/app/config.py` 中的数据库配置：
```python
DB_PASSWORD: str = "12345678"  # 确保密码正确
```

测试数据库连接：
```bash
cd backend
python -c "from app.database import engine; print('数据库连接成功')"
```

## 正常启动流程

1. 双击 `启动系统.bat`
2. 看到"检查后端环境..." → 检查Python和数据库
3. 看到"启动后端服务..." → 打开后端窗口
4. 看到"启动前端服务..." → 打开前端窗口
5. 看到"启动完成！正在打开浏览器..."
6. 浏览器自动打开到 http://localhost:5173
7. 看到登录页面

## 获取更多帮助

如果以上方法都无法解决问题：
1. 查看两个命令窗口的完整错误信息
2. 打开浏览器开发者工具（F12）查看错误
3. 检查浏览器Console和Network标签
4. 记录具体的错误信息以便排查
