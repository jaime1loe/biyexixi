### SQL更新脚本

如果不想删除现有数据，可以手动执行以下SQL：

```sql
-- 为users表添加新字段
ALTER TABLE users
ADD COLUMN phone VARCHAR(20) COMMENT '电话' AFTER email,
ADD COLUMN avatar VARCHAR(255) COMMENT '头像路径' AFTER phone,
ADD COLUMN department VARCHAR(50) COMMENT '院系' AFTER avatar,
ADD COLUMN major VARCHAR(50) COMMENT '专业' AFTER department,
ADD COLUMN class_name VARCHAR(50) COMMENT '班级' AFTER major,
ADD COLUMN is_active INT DEFAULT 1 COMMENT '是否激活' AFTER role,
MODIFY COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;

-- 为questions表添加新字段
ALTER TABLE questions
ADD COLUMN views INT DEFAULT 0 COMMENT '浏览次数' AFTER category,
ADD COLUMN is_public INT DEFAULT 1 COMMENT '是否公开' AFTER views,
ADD COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP AFTER created_at;

-- 为knowledge表添加新字段
ALTER TABLE knowledge
ADD COLUMN file_name VARCHAR(255) COMMENT '文件名' AFTER tags,
ADD COLUMN file_type VARCHAR(50) COMMENT '文件类型' AFTER file_name,
ADD COLUMN file_size INT COMMENT '文件大小' AFTER file_type;

-- 创建收藏表
CREATE TABLE IF NOT EXISTS favorites (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '收藏ID',
    user_id INT NOT NULL COMMENT '用户ID',
    question_id INT NOT NULL COMMENT '问题ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '收藏时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (question_id) REFERENCES questions(id),
    INDEX idx_user_question (user_id, question_id)
) COMMENT='收藏表';

-- 创建通知表
CREATE TABLE IF NOT EXISTS notifications (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '通知ID',
    title VARCHAR(200) NOT NULL COMMENT '通知标题',
    content TEXT COMMENT '通知内容',
    category VARCHAR(50) COMMENT '通知分类',
    is_important INT DEFAULT 0 COMMENT '是否重要',
    file_path VARCHAR(255) COMMENT '附件路径',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '发布时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_category (category),
    INDEX idx_important (is_important)
) COMMENT='通知表';
```

## API端点清单

### 认证和用户
- POST /api/auth/register - 注册
- POST /api/auth/login - 登录
- GET /api/auth/me - 获取当前用户
- PUT /api/auth/me - 更新个人信息
- GET /api/users/ - 获取用户列表（管理员）
- PUT /api/users/{id} - 更新用户信息（管理员）
- DELETE /api/users/{id} - 删除用户（管理员）
- POST /api/users/{id}/reset-password - 重置密码（管理员）

### 问题
- POST /api/questions/ - 提问
- GET /api/questions/ - 获取问题列表
- GET /api/questions/my - 获取我的问题
- GET /api/questions/{id} - 获取问题详情
- PUT /api/questions/{id} - 更新问题分类
- DELETE /api/questions/{id} - 删除问题
- GET /api/questions/export/csv - 导出CSV
- GET /api/questions/categories - 获取分类列表

### 知识库
- POST /api/knowledge/ - 添加知识
- POST /api/knowledge/upload - 上传文件
- GET /api/knowledge/ - 获取知识列表
- GET /api/knowledge/{id} - 获取知识详情
- GET /api/knowledge/{id}/download - 下载文件
- DELETE /api/knowledge/{id} - 删除知识
- GET /api/knowledge/categories - 获取分类列表

### 收藏
- POST /api/favorites/ - 收藏问题
- DELETE /api/favorites/{id} - 取消收藏
- GET /api/favorites/ - 获取收藏列表
- GET /api/favorites/check/{question_id} - 检查是否已收藏

### 通知
- POST /api/notifications/ - 创建通知
- GET /api/notifications/ - 获取通知列表
- GET /api/notifications/{id} - 获取通知详情
- DELETE /api/notifications/{id} - 删除通知
- GET /api/notifications/{id}/download - 下载附件
- GET /api/notifications/categories/list - 获取分类列表

### 校园服务
- GET /api/campus/empty-classrooms - 查询空教室
- GET /api/campus/grades - 查询成绩
- GET /api/campus/library/info - 图书馆信息

## 注意事项

1. **文件上传目录**
   - 知识库文件：`backend/uploads/knowledge/`
   - 通知附件：`backend/uploads/notifications/`
   - 确保这些目录有写入权限

2. **文件大小限制**
   - 默认最大10MB
   - 可在 `.env` 文件中配置 `MAX_FILE_SIZE`

3. **数据库密码**
   - MySQL密码：12345678
   - 确保MySQL服务正在运行

4. **管理员权限**
   - 只有admin角色可以上传文件、删除用户、创建通知等

5. **前端路由**
   - 首页：`/home`
   - 我的收藏：`/favorites`
   - 校园服务：`/campus`
   - 管理后台：`/admin`（仅管理员）
