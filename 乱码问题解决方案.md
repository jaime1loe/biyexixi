# 登录界面乱码问题解决方案

## 问题现象
登录页面显示的中文变成乱码，如"高校知识库智能答疑系统"显示为乱码字符。

## 可能原因
1. **浏览器编码设置错误** - 浏览器未使用UTF-8编码解析页面
2. **文件保存编码错误** - Vue文件保存时使用了非UTF-8编码
3. **服务器响应头缺失** - Vite开发服务器未返回正确的字符编码头
4. **系统字体问题** - 系统缺少中文字体

## 已实施的修复措施

### 1. HTML头部优化（index.html）
```html
<meta charset="UTF-8">
<meta http-equiv="Content-Type" content="text/html; charset=utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
```

### 2. Vite配置优化（vite.config.ts）
```typescript
server: {
  port: 5173,
  headers: {
    'Content-Type': 'text/html; charset=utf-8'
  },
  // ...
},
build: {
  charset: 'utf-8'
}
```

### 3. CSS编码优化（global.css）
```css
html {
  charset: utf-8;
}

body {
  font-family: 'PingFang SC', 'Microsoft YaHei', 'SimHei', 'Arial', sans-serif;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  unicode-bidi: embed;
}

#app {
  unicode-bidi: embed;
}
```

## 用户排查步骤

### 步骤1：清除浏览器缓存并刷新
1. 按 `Ctrl + Shift + Delete` 打开清除缓存窗口
2. 勾选"缓存的图片和文件"
3. 点击"清除数据"
4. 按 `Ctrl + F5` 强制刷新页面

### 步骤2：检查浏览器编码设置
**Chrome浏览器：**
1. 按 `Ctrl + Shift + I` 打开开发者工具
2. 点击 "More tools" → "Encoding"
3. 确保选择 "UTF-8"

**Edge浏览器：**
1. 按 `Ctrl + Shift + I` 打开开发者工具
2. 点击 "设置" → "编码"
3. 确保选择 "Unicode (UTF-8)"

**Firefox浏览器：**
1. 按 `Ctrl + Shift + I` 打开开发者工具
3. 点击 "设置" → "文字编码"
4. 确保选择 "Unicode (UTF-8)"

### 步骤3：重启前端服务
```bash
# 停止当前服务（在命令窗口按 Ctrl+C）
# 然后重新启动
cd frontend
npm run dev
```

### 步骤4：清除node_modules并重新安装
```bash
cd frontend
# Windows
rmdir /s /q node_modules

# 重新安装依赖
npm install --registry=https://registry.npmmirror.com

# 重新启动
npm run dev
```

### 步骤5：检查系统字体
确保系统已安装中文字体：
- Windows：Microsoft YaHei（微软雅黑）、SimHei（黑体）
- macOS：PingFang SC、STHeiti

如果字体缺失，请下载并安装中文字体包。

### 步骤6：尝试其他浏览器
如果当前浏览器仍有问题，尝试使用：
- Chrome
- Edge
- Firefox
- Safari（仅macOS）

## 终极解决方案

如果以上方法都无法解决，请执行以下步骤：

### 1. 重新生成Vue文件（使用UTF-8编码保存）
确保所有Vue文件以UTF-8 without BOM编码保存：
- VSCode：右下角点击编码 → "Save with Encoding" → "UTF-8"
- Notepad++：编码 → "转为UTF-8无BOM编码"

### 2. 修改package.json添加字符集声明
```json
{
  "type": "module",
  "scripts": {
    "dev": "vite --charset=utf-8",
    // ...
  }
}
```

### 3. 添加环境变量
在项目根目录创建 `.env` 文件：
```
VITE_CHARSET=utf-8
```

### 4. 重启开发服务器
```bash
cd frontend
npm run dev
```

## 预防措施

1. **使用现代IDE**：推荐使用VSCode、WebStorm等编辑器，确保文件以UTF-8编码保存

2. **配置编辑器默认编码**：
   - VSCode：设置 `files.encoding` 为 `utf8`
   - WebStorm：设置 "File Encodings" 为 UTF-8

3. **避免使用特殊字符**：在代码中尽量避免使用生僻字和特殊符号

4. **定期检查**：开发过程中定期检查页面编码是否正确
