## 二、后端开发进度

### 2.1 语法错误修复

**问题描述**：后端启动时出现 `SyntaxError: expected 'except' or 'finally' block` 错误

**问题原因**：
1. `backend/app/routers/statistics.py` 中 `get_top_questions` 函数的 `try` 语句缩进错误，导致代码结构不完整
2. 导入路径问题：`database.py` 和 `models.py` 使用了错误的绝对导入路径

**修复内容**：
| 文件 | 修改内容 |
|------|----------|
| `backend/app/database.py` | 将 `from backend.app.config import settings` 改为 `from app.config import settings` |
| `backend/app/models.py` | 将 `from backend.app.database import Base` 改为 `from app.database import Base` |
| `backend/app/routers/statistics.py` | 修复 `get_top_questions` 函数缩进，确保 try-except 块结构完整 |

### 2.2 数据库字段完善

**问题描述**：数据库操作失败，500 Internal Server Error

**问题原因**：数据库 `questions` 表缺少 `ask_count` 字段

**修复内容**：
- 在 `Question` 模型中添加 `ask_count` 字段
- 执行 SQL 语句为数据库添加该字段

```python
# backend/app/models.py
ask_count = Column(Integer, default=1, comment="提问次数")
```

### 2.3 提问功能增强

**问题描述**：需要实现重复提问计数功能

**修复内容**：
- 修改 `backend/app/routers/questions.py` 中的 `create_question` 接口
- 实现逻辑：如果问题已存在，增加 `ask_count` 次数；否则创建新问题

```python
# 检查是否已有相同问题
existing_question = db.query(Question).filter(
    Question.question == question.question
).first()

if existing_question:
    # 如果问题已存在，增加提问次数
    existing_question.ask_count += 1
    db.commit()
    return existing_question
else:
    # 创建新问题记录
    db_question = Question(...)
```

### 2.4 统计功能增强

**问题描述**：数据统计看板中总提问数没有显示

**问题原因**：后端统计 API 返回字段与前端定义不匹配

**修复内容**：
1. 增强 `get_statistics_overview` 接口，新增返回字段：
   - `answer_count`：总回答数（有答案的问题数量）
   - `ask_count`：总提问次数（所有问题的 ask_count 总和）

2. 修改热门问题排序逻辑：
   - 从按创建时间倒序改为按提问次数（ask_count）倒序

```python
# backend/app/routers/statistics.py
questions = db.query(Question).order_by(
    desc(Question.ask_count)
).limit(limit).all()
```

### 2.5 Schema 完善

**问题描述**：API 响应缺少 `ask_count` 字段

**修复内容**：在 `QuestionResponse` 模型中添加 `ask_count` 字段

```python
# backend/app/schemas.py
class QuestionResponse(QuestionBase):
    ask_count: Optional[int] = 1
```
