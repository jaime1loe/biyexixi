## 二、数据库模型

### 2.1 数据表结构

系统共包含 **7张核心数据表**：

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `users` | 用户表 | username, password_hash, role, student_id, email |
| `questions` | 问题表 | question, answer, category, views, ask_count |
| `feedbacks` | 反馈评价表 | rating(1-5星), comment |
| `knowledge` | 知识库表 | title, content, category, tags, review_status |
| `statistics` | 统计表 | date, question_count, user_count, avg_rating |
| `favorites` | 收藏表 | user_id, question_id |
| `notifications` | 通知表 | title, content, category, is_important |

### 2.2 用户角色

系统支持三种用户角色：
- **student（学生）**：普通用户，可提问、查看知识库、提交反馈
- **teacher（教师）**：可管理知识库、审核内容
- **admin（管理员）**：最高权限，可管理用户、审核知识、发布通知

## 四、安全特性

### 4.1 认证与授权

- **JWT Token 认证**：基于 Bearer Token 的无状态认证
- **密码加密**：使用 bcrypt 进行密码哈希存储
- **角色权限控制**：基于角色的访问控制（RBAC）

### 4.2 数据验证

- **Pydantic 数据模型**：请求数据的自动验证
- **必填字段校验**：如用户名、密码长度等
- **邮箱格式验证**：使用 EmailStr 类型

### 4.3 异常处理

- **统一异常处理器**：SQLAlchemy、HTTP、验证异常统一处理
- **自定义错误信息**：友好的错误提示
