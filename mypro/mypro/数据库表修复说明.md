# 数据库表修复说明

## 修复时间
2026年3月11日

## 问题描述
启动系统后访问首页时，出现以下错误：
1. 通知API返回500错误
2. 问题API返回500错误
3. 主界面各个功能无法连接数据库加载失败

## 根本原因
数据库中缺少必要的表和字段：
- `notifications` 表不存在
- `questions` 表缺少 `views`、`is_public` 和 `updated_at` 字段
- `knowledge` 表缺少 `file_name`、`file_type`、`file_size`、`embedding` 字段
- `favorites` 表不存在

## 修复步骤

### 1. 创建缺失的数据库表
执行 `init_tables.py` 创建所有缺失的表：
- `notifications` (通知表)
- `favorites` (收藏表)

### 2. 修复 questions 表结构
执行 `fix_questions_table.py` 添加缺失的字段：
- `views` (浏览次数，默认0)
- `is_public` (是否公开，默认1)
- `updated_at` (更新时间)

### 3. 修复 knowledge 表结构
执行 `fix_all_tables.py` 添加缺失的字段：
- `file_name` (文件名)
- `file_type` (文件类型)
- `file_size` (文件大小)
- `embedding` (向量数据)

### 4. 简化数据模型
修改 `schemas.py`，移除复杂的嵌套关系：
- `QuestionResponse` 不再包含完整的 `user` 对象
- `FeedbackResponse` 不再包含嵌套的 `user` 和 `question` 对象
- `FavoriteResponse` 不再包含嵌套的 `question` 对象

### 5. 添加示例数据
执行 `add_sample_data.py` 添加示例通知数据：
- 系统维护通知
- 新功能上线通知
- 校园服务功能优化通知

3. 统一统计数据的字段名
4. 修复 ElementPlus 组件的废弃警告

## 启动说明
现在系统应该可以正常运行：
1. 双击 `backend\运行后端.bat` 启动后端
2. 双击 `frontend\运行前端.bat` 启动前端
3. 访问 http://localhost:5173 即可进入系统

如需重新登录，使用以下方法之一：
- 点击顶部导航栏用户头像 → 退出登录
- 访问 http://localhost:5173/?force=true 强制清除登录状态
- 在浏览器控制台执行 `localStorage.clear(); location.reload()`
