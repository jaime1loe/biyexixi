## 项目概述

构建基于大模型智能体的高校知识库在线答疑系统，为高校学生提供7×24小时智能答疑服务，整合校内知识资源，实现"问答到办事"的业务闭环。

## 核心功能需求

- 智能工单引擎：自动识别可办理事项，一句话生成工单
- 多智能体协同矩阵：学术支持、奖学金查询、就业指导等专项智能体
- 进度追踪与可视化：工单全生命周期透明可追踪
- 校园知识图谱：自动提取实体与关系，支持复杂逻辑推理
- GraphRAG智能问答：向量相似度检索与图谱逻辑遍历
- 启发式学习引导：通过思维链拆解引导学生自主思考
- 错因分析与个性化推荐
- 心理健康感知
- 多维评价体系
- 服务热力图
- 知识库自优化
- 私有化部署方案

## 项目约束

- 部署方式：纯本地部署（私有化）
- 大模型：开源大模型（ChatGLM/Qwen等）
- 功能范围：全部16个功能
- 实现程度：可演示的原型系统

## 技术栈选型

- 后端：Java (SpringBoot 2.7 + MyBatis + MySQL 8.0)
- 前端：Vue 3 + Element Plus + Axios
- AI开发：Python 3.10 + LangChain + RAG/GraphRAG
- 大模型：ChatGLM3-6B（开源本地部署）
- 向量数据库：Milvus 或 FAISS
- 知识图谱：Neo4j

## 系统架构设计

### 整体架构

采用前后端分离 + AI服务分离的分布式架构：

- 前端服务：Vue 3 单页应用，提供用户交互界面
- 后端服务：SpringBoot REST API，处理业务逻辑和数据持久化
- AI服务：Python FastAPI，提供大模型问答和知识检索能力
- 数据库：MySQL（业务数据）+ 向量数据库（知识库）+ Neo4j（图谱数据）

### 模块划分

1. **用户端模块**：问答界面、工单提交、进度查询、个人中心
2. **管理端模块**：知识库管理、工单管理、数据看板、用户管理
3. **AI核心模块**：RAG检索、知识图谱、多智能体编排、答案生成

## 设计风格

采用现代简洁的教育类产品设计风格，强调专业感与亲和力的平衡。界面以浅色系为主，搭配主题蓝色营造科技感，整体布局清晰直观，符合高校师生的审美和使用习惯。

## 页面规划

1. **首页/问答页**：主要交互界面，包含问题输入框、对话列表、快捷入口
2. **工单中心页**：展示提交的工单列表、状态追踪、详情查看
3. **个人中心页**：用户信息、学习记录、收藏内容、提问历史
4. **管理后台**：知识库管理、工单管理、数据看板、用户管理

## 设计要素

- 主色调：#1890FF（科技蓝）
- 辅助色：#52C41A（成功）、#FAAD14（警告）、#F5222D（错误）
- 背景色：#F5F7FA（浅灰背景）、#FFFFFF（卡片背景）
- 字体：思源黑体 / PingFang SC
- 间距：8px网格系统
- 圆角：8px（卡片）、4px（按钮）
