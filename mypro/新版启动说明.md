# 新版启动方法说明

## 问题诊断

原启动脚本失败的可能原因：
1. 路径设置复杂导致错误
2. 多层命令嵌套导致解析失败
3. 编码设置在某些情况下不生效

## 新版启动方案

### 方案A：新版一键启动（推荐）

双击运行：`启动系统-新版.bat`

**特点：**
- ✅ 最简洁的命令结构
- ✅ 直接使用 Python 和 npm 命令
- ✅ 无复杂嵌套，兼容性最好
- ✅ 自动打开两个命令窗口

---

### 方案B：分别启动（最可靠）

**第一步：启动后端**
```
双击运行：backend/运行后端.bat
```
等待看到：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```
**不要关闭这个窗口！**

**第二步：启动前端**
```
双击运行：frontend/运行前端.bat
```
等待看到：
```
Local:   http://localhost:5173/
```
**不要关闭这个窗口！**

**第三步：打开浏览器**
```
访问：http://localhost:5173
```

---

### 方案C：手动命令行启动（最可控）

**启动后端：**
```bash
cd backend
python main.py
```

**启动前端：**
```bash
cd frontend
npm run dev
```

## 新版启动脚本特点

### `backend/运行后端.bat`
```batch
@echo off
cd /d "%~dp0"
echo Starting backend server...
python main.py
pause
```
- 仅4行代码
- 切换到脚本所在目录
- 直接运行 main.py
- 无任何复杂操作

### `frontend/运行前端.bat`
```batch
@echo off
cd /d "%~dp0"
echo Starting frontend server...
npm run dev
pause
```
- 仅4行代码
- 切换到脚本所在目录
- 直接运行 npm run dev
- 无任何复杂操作

### `启动系统-新版.bat`
```batch
@echo off
chcp 65001 >nul

# 启动后端
start "后端服务" cmd /k "cd backend && python main.py"

# 等待8秒
timeout /t 8

# 启动前端
start "前端服务" cmd /k "cd frontend && npm run dev"

# 等待5秒
timeout /t 5

# 打开浏览器
start http://localhost:5173
```
- 简化的命令结构
- 每个步骤独立执行
- 无复杂嵌套

3. 自动打开浏览器
4. 完成！

## 常见问题

### Q1: 双击后窗口闪退

**解决：**
- 右键点击 `.bat` 文件
- 选择"以管理员身份运行"
- 或者直接从命令提示符运行

### Q2: 显示"python不是内部或外部命令"

**解决：**
- 重新安装 Python
- 安装时勾选 "Add Python to PATH"
- 重启电脑后重试

### Q3: 显示"npm不是内部或外部命令"

**解决：**
- 安装 Node.js
- 下载地址：https://nodejs.org/
- 重启电脑后重试

### Q4: 显示数据库连接错误

**解决：**
- 确认 MySQL 服务正在运行
- 确认密码是 20040618
- 运行 `backend/test_db_connection.py` 测试

### Q5: 前端端口被占用

**解决：**
- 方法1：关闭其他使用5173端口的程序
- 方法2：修改 `frontend/vite.config.ts` 中的端口

## 停止服务

**正确方式：**
1. 在后端窗口按 `Ctrl+C`
2. 在前端窗口按 `Ctrl+C`
3. 等待两个窗口都显示停止信息
4. 关闭窗口

**错误方式（不要这样做）：**
- ❌ 直接点击窗口的 X 按钮
- ❌ 使用任务管理器强制关闭

## 技术支持

如果以上方法都无法解决问题，请：

1. 查看命令窗口的错误信息
2. 截图保存错误信息
3. 检查系统环境：
   - Python版本：运行 `python --version`
   - Node.js版本：运行 `node --version`
   - MySQL服务：运行 `services.msc` 查看MySQL状态
4. 尝试手动命令行启动获取更详细错误
