# 排课管理导入错误修复说明

## 问题描述

在实现排课管理功能时，前端编译报错：
```
Failed to resolve import "@/utils/request" from "src/api/schedule.ts"
```

## 问题原因

项目原本的API调用使用的是 `frontend/src/api/index.ts` 中定义的 `request` 实例，而不是 `@/utils/request`。

## 解决方案

### 1. 修复API导入路径

**文件**: `frontend/src/api/schedule.ts`

**修改前**:
```typescript
import request from '@/utils/request'
```

**修改后**:
```typescript
import request from './index'
```

### 2. 创建课程管理API文件

**文件**: `frontend/src/api/course.ts`

创建了完整的课程管理API接口，包括：
- `getCourses` - 获取课程列表
- `getCourse` - 获取课程详情
- `createCourse` - 创建课程
- `updateCourse` - 更新课程
- `deleteCourse` - 删除课程
- `getTeachers` - 获取教师列表

### 3. 调整API路径

后端 `course_management` 路由的路径结构：
```
POST   /api/course-management              - 开设课程
GET    /api/course-management              - 查询所有课程
GET    /api/course-management/{id}        - 查询课程详情
PUT    /api/course-management/{id}        - 更新课程
DELETE /api/course-management/{id}        - 删除课程
GET    /api/course-management/teachers/list - 获取教师列表
```

前端API路径已调整为匹配后端路由。

## 重要提示

1. **导入路径**: 项目统一使用 `./index` 导入 request，而不是 `@/utils/request`

2. **API路径**: 所有API调用都应该通过 `index.ts` 中的 request 实例，确保拦截器正常工作

3. **响应拦截**: `index.ts` 中的响应拦截器会自动提取 `response.data`，所以调用API时返回的已经是数据本身

4. **错误处理**: 响应拦截器会自动处理常见错误（401、403、404、500），并显示ElMessage
