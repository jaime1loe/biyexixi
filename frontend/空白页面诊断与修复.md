# 空白页面诊断与修复

## 🔍 可能的原因

### 1. 依赖未安装(最常见)
```
症状: 页面空白,浏览器控制台显示 "404 Not Found"
```

### 2. 路由配置问题
```
症状: 页面空白,控制台无错误
```

### 3. Element Plus 未正确加载
```
症状: 页面空白,控制台显示组件未注册错误
```

### 4. Vue 应用未挂载
```
症状: 完全空白,甚至连背景色都没有
```

---

## ✅ 解决步骤

### 第一步: 确认依赖已安装

打开命令提示符,执行:

```bash
cd c:\Users\19719\Desktop\2026毕业实习\mypro\frontend
```

检查 `node_modules` 文件夹是否存在:
```bash
dir
```

**如果没有 `node_modules` 文件夹,需要安装依赖:**

```bash
npm install --registry=https://registry.npmmirror.com
```

等待安装完成(需要2-3分钟),看到类似以下信息:
```
added 852 packages in 2m 30s
```

---

### 第二步: 重新启动开发服务器

如果服务器正在运行,先停止:
- 按 `Ctrl + C`

然后重新启动:
```bash
npm run dev
```

看到以下信息说明启动成功:
```
VITE v5.0.0  ready in 234 ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

---

### 第三步: 清除浏览器缓存

1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存的图片和文件"
3. 点击"清除数据"
4. 刷新页面 (F5)

或者使用硬刷新:
- Windows: `Ctrl + F5`
- Mac: `Cmd + Shift + R`

---

### 第四步: 检查浏览器控制台

1. 按 `F12` 打开开发者工具
2. 切换到 "Console" 标签
3. 查看是否有错误信息

#### 常见错误及解决

**错误1: "Failed to resolve import"**
```
原因: 依赖未安装
解决: npm install
```

**错误2: "404 Not Found"**
```
原因: 文件路径错误
解决: 确认从 frontend 目录启动
```

**错误3: "Cannot find module"**
```
原因: 某个依赖包缺失
解决: npm install <缺失的包名>
```

## 🔧 完整重置步骤

如果以上都不行,执行完整重置:

### 1. 停止开发服务器
```bash
Ctrl + C
```

### 2. 删除 node_modules 和 package-lock.json
```bash
rmdir /s /q node_modules
del package-lock.json
```

### 3. 重新安装依赖
```bash
npm install --registry=https://registry.npmmirror.com
```

### 4. 重新启动
```bash
npm run dev
```

## 🐛 如果还是空白

### 尝试更换端口

编辑 `vite.config.ts`,修改端口:

```typescript
server: {
  port: 5174,  // 改成其他端口
  // ...
}
```

然后重新启动:
```bash
npm run dev
```

访问新的地址: `http://localhost:5174`

---

### 检查文件完整性

确认以下文件存在:

```
frontend/
├── index.html          # 必须有

├── tsconfig.json       # 必须有
├── src/
│   ├── main.ts         # 必须有
│   ├── App.vue         # 必须有
│   ├── router/

│   └── views/
```
